Mapping data to elements

You start with a HTML-structure that has to be filled in:

<div id="order-notification"> <h2></h2> <p></p> </div>

This is the data you have, usually retrieved with an AJAX-call:

var data = { "title": "Order status update", "content": "Your order is ready to ship." };

This is the map, which maps the data to the DOM-elements. The h2-element is mapped to the title-property in the data and the p-element is mapped to the content-propertie in the data.

var map = { "h2": "title", "p": "content" };

Now just calling the maskr-plugin with the map- and data-variables will merge the data in the DOM:

$("#order-notification").maskr(map, data);

And the blanks are filled in:

<div id="order-notification"> <h2>Order status update</h2> <p>Your order is ready to ship.</p> </div>

Mapping data to properties

By using the @-symbol, you can set the properties of elements. This is the DOM-template:

<div id="order-notification"> <h2></h2> <p></p> <a>more info</a> </div>

The data:

var data = { "title": "Order status update", "content": "Your order is ready to ship.", "type": "alert", "info-url": "/notification-info?193" };

This is the map, which maps the data to the DOM-elements and -properties. You can use a selector in front of the @-sign. If you don't use a selector in front of the @-sign, the root element is selected.

var map = { "h2": "title", "p": "content", "@class": "type", "a@href": "info-url" };

Now just calling the maskr-plugin with the map- and data-variables will merge the data with the DOM:

$("#order-notification").maskr(map, data);

And the blanks are filled in:

<div class="alert" id="order-notification"> <h2>Order status update</h2> <p>Your order is ready to ship.</p> <a href="/notification-info?193">more info</a> </div>

Replacing text

You can also replace certain parts in a text. Just mark the placeholders with a name between curly brackets:

<p id="greeting">Hello {name}, you have {msgnr} messages left.</p>

The data:

var data = { "fullname": "John Smith", "messages": "14" };

Enclose the keys in curly brackets to make them replace the placeholders.

var map = { "{name}": "fullname", "{msgnr}": "messages" };

Calling the maskr-plugin

$("#greeting").maskr(map, data);

Results in this greeting:

<p id="greeting">Hello John Smith, you have 14 messages left.</p>

Replacing text in attributes.

As you can see in the example below, you can use placeholders in DOM-properties too:

<div class="notification {importance}" id="order-notification"> <h2>Order status update for {name}</h2> <a href="/order-info?nr={ordernr}">more info about order {ordernr}</a> </div>

The data:

var data = { "imp": "alert", "fullname": "John Smith", "oid": "834" };

Enclose the placeholdernames in curly brackets to make them replace the placeholders.

var map = { "@class{importance}": "imp", "h2{name}": "fullname", "a@href{ordernr}": "oid", "a{ordernr}": "oid" };

Calling the maskr-plugin

$("#order-notification").maskr(map, data);

Results in this notification:

<div class="notification alert" id="order-notification"> <h2>Order status update for John Smith</h2> <a href="/order-info?nr=834">more info about order 834</a> </div>

Simple arrays

Data is not always as one-dimensional as in the examples above. Often, you'll have datastructures containing arrays. The simplest form is an array of strings.

Let's start with this simple, empty list:

<ul id="names"> <li></li> </ul>

This is the data we want to display:

{ "friends": ["John", "Paul", "Gerard"] }

With this map, we couple the li-element to the friends-array. The friends-string is the content of a one-element array.

{ "li": ["friends"] }

The result of merging them is that the li-element is repeated for every element in the friends-array:

<ul id="names"> <li>John</li> <li>Paul</li> <li>Gerard</li> </ul>

Complex arrays

Often, datastructures are more complex and can consist of arrays of other datastructures.

For example, you want to show the names of friends with an informational message below.

<ul id="names"> <li> <h2></h2> <p></p> </li> </ul>

This is the data:

{ "friends": [ { "name": "John", "mesg": "no messages" }, { "name": "Paul", "mesg": "3 messages" }, { "name": "Gerard", "mesg": "1 message" } ] }

Just like in the example above, we couple the li-elements to the friends-array. But now the one-element array gets a second element: a new map for the li-elements. This is recursive: you can repeat this and go as deep as you want.

{ "li": ["friends", { "h2": "name", "p": "mesg" } ] }

The result:

<ul id="names"> <li> <h2>John</h2> <p>no messages</p> </li> <li> <h2>Paul</h2> <p>3 messages</p> </li> <li> <h2>Gerard</h2> <p>1 message</p> </li> </ul>

Functions

Sometimes you want to process the data before inserting it. You can do this with custom functions.

<ul id="names"> <li> <a href="/books/{shortname}"></a> </li> </ul>

The data is a list of book titles:

{ "books": [ { "title": "Pro JavaScript Techniques" }, { "title": "JavaScript the Good Parts" }, { "title": "Pro JavaScript Design Patterns" } ] }

In this example, we use a custom function two times. The function receives two parameters: the data-object and the index, starting with zero. In the first function, we use the index to number the book titles. In the second function, we conveRt the title to lowercase and replace spaces with minus-signs and url-encode it, so we end up with a nice url to the book.

{ "li": [ "books", { "a": function(data, i) {return (i + 1) + ". " + data.title}, "a@href{shortname}": function(data, i) {return encodeURI(data.title.replace(/ /g, '-').toLowerCase())} } ] }

The result after running maskr:

<ul id="books"> <li> <a href="/books/pro-javascript">1. Pro JavaScript Techniques</a> </li> <li> <a href="/books/javascript-the-good-parts">2. JavaScript the Good Parts</a> </li> <li> <a href="/books/javascript-design-patterns">3. Pro JavaScript Design Patterns</a> </li> </ul>

Inside the custom function, this refers to the DOM-element.

String functions

Maskr supports a special notation to apply string functions. It looks like:

prop|function:arg1:arg2...

You can use any String function you want: toUpperCase, replace, charAt etcetera. The functionality is limited in that the arguments can only be literal strings.

We start with a very simple template:

<ul id="names"> <li>{title}</li> </ul>

The data is the same list of book titles as in the previous example:

{ "books": [ { "title": "Pro JavaScript Techniques" }, { "title": "JavaScript the Good Parts" }, { "title": "Pro JavaScript Design Patterns" } ] }

Now we use the toUpperCase string function:

{ "li": [ "books", { "{title}": "title|toUpperCase" } ] }

The result after running maskr:

<ul id="names"> <li>PRO JAVASCRIPT TECHNIQUES</li> <li>JAVASCRIPT THE GOOD PARTS</li> <li>PRO JAVASCRIPT DESIGN PATTERNS</li> </ul>

You can also chain string functions. Let's take the uppercase function and add a substr function:

{ "li": [ "books", { "{title}": "title|toUpperCase|substr:11:5" } ] }

This will convert the title to uppercase and returns the substring from character 11 with a length of 5 characters.

<ul id="names"> <li>IPT T</li> <li>THE G</li> <li>IPT D</li> </ul>

String functions that are useful to use:

charAtReturns the character at the specified location.
concatConcats a string to the string.
indexOfReturns the index of a substring or -1 when not found.
lastIndexOfReturns the last index of a substring or -1 when not found.
localeCompareCompares the string with another string.
replaceReplace a substring with another.
sliceGets a part of the string.
substrGets a part of the string.
substringGets a part of the string.
toLowerCaseConverts de string to lowercase.
toUpperCaseConverts de string to uppercase.
trim(JavaScript 1.8.1+) Removes white space on both sides of the string.
trimLeft(JavaScript 1.8.1+) Removes white space on the left side of the string.
trimRight(JavaScript 1.8.1+) Removes white space on the right side of the string.

Maskr has three build-in functions: html, urlEncode and if.

html

Normally, if the data contains HTML-markup, the markup is shown as is. Sometimes, your data has HTML-markup and you want to have the markup applied. When you want to apply the markup, just at the "html" string function:

{ "li": [ "books", { "{title}": "title|toUpperCase|substr:11:5|html" } ] }

Now, if the title has a subtitle between <i>-tags, it will be displayed in italics.

urlEncode

Apply the urlencode-function. Usefull for generating URL's.

Let's take the map from the functions chapter:

{ "li": [ "books", { "a": function(data, i) {return (i + 1) + ". " + data.title}, "a@href{shortname}": function(data, i) {return encodeURI(data.title.replace(/ /g, '-').toLowerCase())} } ] }

This might also be written in the equivalent but better readable:

{ "li": [ "books", { "a": function(data, i) {return (i + 1) + ". " + data.title}, "a@href{shortname}": "title|replace: :-:g|toLowerCase|urlEncode" } ] }

The if-function gives you the tertiary operator val == a ? b : c. Dependent of the value, you can output one or another value.

For example, you can combine it with the indexOf function. Let's take the example template again:

<ul id="names"> <li>{title}</li> </ul>

The data is the same list of book titles:

{ "books": [ { "title": "Pro JavaScript Techniques" }, { "title": "JavaScript the Good Parts" }, { "title": "Pro JavaScript Design Patterns" } ] }

Now we search the title for "Design Patterns". If the string is not found, indexOf returns -1 and we give it the non-design-patterns class. If the string is found, we give it the design-patterns class.

{ "li": [ "books", { "{title}": "title", "@class": "title|indexOf:Design Patterns|if:-1:non-design-patterns:design-patterns" } ] }

The result after running maskr:

<ul id="names"> <li class="non-design-patterns">Pro JavaScript Techniques</li> <li class="non-design-patterns">JavaScript the Good Parts</li> <li class="design-patterns">Pro JavaScript Design Patterns</li> </ul>
