Phases of a JavaScript developer

  1. Why do I need a library?
  2. Starts writing functions for common, tedious tasks.
  3. Handfull of functions becomes a library of functions.
  4. Discovers that another library does the same, but better on all fronts.

Why a JavaScript library?

  • Masks browser incompatibilities
  • Prevents memory leaks
  • Optimised for speed and size
  • Maintained by specialists

Which library?

  • Functionality (widgets? framework?)
  • Speed
  • Size (bytes)
  • Documentation
  • Community
  • Extensible
  • Use of globals
  • License (Open Source)

Easing

  • c*t/d+b
  • c*((t=t/d-1)*t*t + 1) + b
  • -c * Math.cos(t/d * (Math.PI/2)) + c + b
  • c * Math.sqrt(1 - (t=t/d-1)*t) + b
  • c*(t/=d)*t*t*t*t + b
  • c*t/d*((Math.cos(t/d * (Math.PI/2))+1))*1.1-0.1+b
