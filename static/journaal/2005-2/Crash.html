Crash

do 10 feb 2005 23:35

Gisteren is de harddisk van m'n server gecrasht. Eerst dacht ik nog dat de server gehackt was. Sommige pagina's waren wel te zien en andere niet. Alsof iemand gericht pagina's had gewist. Na het aansluiten van een monitor op de server werd snel duidelijk wat er aan de hand was. De disk-lees-foutmeldingen vlogen over het scherm. Na een herstart was de disk helemaal niet meer te bereiken.

Thuis heb ik er een nieuw moederbord in gezet (het moederbord dat over was nadat ik m'n werkcomputer had geupgrade). De SCSI-harddisks zijn vervangen door "gewone" IDE harddisks (met SCSI heb ik tot nu toe enkel problemen gehad, maar dat is een ander verhaal). Debian Linux erop gezet. Apache geïnstalleerd. Nu nog een backup terugzetten. Hmm. De laatste backup van de complete server is van december 2003. Dit betekent dat ik ruim een jaar van m'n weblog kwijt ben. Ook L.'s "weblog" heeft een even groot gat. Ja, ik ben zo stom geweest om van de weblogs geen backups te maken... (Jolie was overigens wel zo slim).

En de kapotte harddisk? Die kan ik heel waarschijnlijk wel vergeten. Vanmiddag heb ik allerlei tooltjes uitgeprobeerd en allemaal konden ze niet één byte lezen. Als iemand nog tips heeft om kapotte disks tot leven te wekken? (M'n zwager Mark heeft al hulp aangeboden).

Er is één klein voordeel: de server heeft nu iets snellere hardware. Ondertussen ben ik m'n oude weblog-stukjes uit de Google-cache aan het vissen.
